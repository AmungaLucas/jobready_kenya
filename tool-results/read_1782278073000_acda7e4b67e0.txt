     7→# 🧠 Candidate-to-Job Matching Engine Documentation
     8→
     9→## Version: Phase 2 Technical Architecture (v2.0 – Comprehensive)
    10→
    11→---
    12→
    13→## Core Principle
    14→
    15→The platform uses the following principle:
    16→
    17→**AI-on-Ingest, CPU-on-Match**
    18→
    19→This means:
    20→
    21→- AI or parser logic is used when a CV or job is uploaded.
    22→- The uploaded CV or job post is converted into structured data.
    23→- Structured data is saved in MySQL.
    24→- Matching is done using SQL joins and scoring logic.
    25→- AI is not used repeatedly during matching.
    26→- CV files are not permanently stored.
    27→
    28→This keeps the platform scalable, cheap, and fast.
    29→
    30→---
    31→
    32→## 1. Product Vision
    33→
    34→The platform helps candidates discover jobs that match their real skills, qualifications, experience, and career interests.
    35→
    36→The goal is not to match only job titles.
    37→
    38→The goal is to understand:
    39→
    40→- What the candidate can do
    41→- What the job requires
    42→- What category of work the role belongs to
    43→- What subcategory or specialization the role belongs to
    44→- What skills are required
    45→- What qualifications are required
    46→- What the candidate is interested in
    47→- What jobs are realistic for the candidate
    48→
    49→---
    50→
    51→## 2. Matching Philosophy
    52→
    53→The system separates:
    54→
    55→- Organization type
    56→- Organization industry
    57→- Job category
    58→- Job subcategory
    59→- Role title
    60→- Skills
    61→- Tools
    62→- Qualifications
    63→- Experience
    64→- Candidate preferences
    65→
    66→### Example
    67→
    68→Kenyatta National Hospital is in the healthcare industry.
    69→
    70→But if they hire an HR Officer:
    71→
    72→- Organization industry: Healthcare
    73→- Job category: Human Resources
    74→- Job subcategory: HR Generalist or Recruitment
    75→- Role: HR Officer
    76→
    77→The candidate does not need to be a healthcare worker.
    78→
    79→They need to be an HR professional.
    80→
    81→Therefore:
    82→
    83→Industry is context.
    84→
    85→Category and subcategory are strong matching signals.
    86→
    87→Skills and qualifications are proof.
    88→
    89→---
    90→
    91→## 3. Why Industry Has Low Matching Weight
    92→
    93→A professional can work across many industries.
    94→
    95→Example:
    96→
    97→An HR Officer can work in:
    98→
    99→- Healthcare
   100→- Banking
   101→- Telecommunications
   102→- Education
   103→- Government
   104→- Manufacturing
   105→- NGO
   106→
   107→A Finance Officer can work in:
   108→
   109→- NGO
   110→- Government
   111→- Healthcare
   112→- Banking
   113→- Manufacturing
   114→- Education
   115→
   116→So industry should not dominate matching.
   117→
   118→Recommended use:
   119→
   120→- Industry familiarity is a small bonus.
   121→- It should not reject otherwise good candidates.
   122→- Category, subcategory, skills, and qualifications should carry more weight.
   123→
   124→---
   125→
   126→## 4. Final Matching Weights
   127→
   128→Recommended scoring model:
   129→
   130→```js
   131→const MATCH_WEIGHTS = {
   132→  categoryMatch: 25,
   133→  subcategoryMatch: 25,
   134→  skillsMatch: 25,
   135→  qualificationsMatch: 10,
   136→  experienceMatch: 10,
   137→  industryFamiliarity: 5
   138→};
   139→```
   140→
   141→Total: 100%
   142→
   143→Meaning:
   144→
   145→- Category match checks if the candidate is in the same career function.
   146→- Subcategory match checks specialization.
   147→- Skills match checks capability.
   148→- Qualifications match checks eligibility.
   149→- Experience match checks level.
   150→- Industry familiarity gives a small contextual bonus.

   154→## 5. System Architecture
   155→
   156→### Candidate Upload Flow
   157→
   158→```txt
   159→Candidate uploads CV
   160→        ↓
   161→Backend receives file in memory
   162→        ↓
   163→Markitdown extracts Markdown/text
   164→        ↓
   165→Parser extracts structured data
   166→        ↓
   167→Optional AI fallback if parser confidence is low
   168→        ↓
   169→Normalize data against taxonomy
   170→        ↓
   171→Save structured profile to MySQL
   172→        ↓
   173→Discard uploaded file
   174→        ↓
   175→Run CPU-based matching
   176→        ↓
   177→Save top matches
   178→        ↓
   179→Return candidate profile + matches
   180→```
   181→
   182→---
   183→
   184→### Job Upload Flow
   185→
   186→```txt
   187→Recruiter uploads job post
   188→        ↓
   189→Backend receives job text
   190→        ↓
   191→Parser or AI extracts structured job profile
   192→        ↓
   193→Normalize against taxonomy
   194→        ↓
   195→Save job and job profile to MySQL
   196→        ↓
   197→Run CPU-based matching against candidates
   198→        ↓
   199→Save candidate-job scores
   200→```

   204→## 6. No Media Storage Rule
   205→
   206→CV files should not be permanently stored.
   207→
   208→The file should be:
   209→
   210→1. Uploaded.
   211→2. Processed in memory.
   212→3. Converted to text.
   213→4. Discarded.
   214→
   215→The database stores:
   216→
   217→- Extracted name
   218→- Email
   219→- Phone
   220→- Skills
   221→- Education
   222→- Work experience
   223→- Qualifications
   224→- Career category
   225→- Subcategories
   226→- Matching data
   227→
   228→The database should not store the original CV file unless a future business requirement requires it.
   229→
   230→Optional:
   231→
   232→You may store raw extracted CV text for debugging for a short period, for example 30 days.

   236→## 7. AI Role in the System
   237→
   238→AI should not be the main matcher.
   239→
   240→AI is used for:
   241→
   242→### 7.1 CV Extraction
   243→

   245→
   246→Example:
   247→
   248→```txt
   249→Finance Intern at PATH Kenya
   250→Managed Accounts Payable, reconciliations, audit schedules...
   251→```
   252→
   253→Becomes:
   254→
   255→```json
   256→{
   257→  "category": "FINANCE_ACCOUNTING",
   258→  "subcategories": ["ACCOUNTING", "ACCOUNTS_PAYABLE", "AUDIT"],
   259→  "skills": ["ACCOUNTS_PAYABLE", "BANK_RECONCILIATION", "AUDIT_SUPPORT"],
   260→  "tools": ["ERP_SYSTEMS", "EXCEL"],
   261→  "industriesWorkedIn": ["INTERNATIONAL_DEVELOPMENT", "NON_PROFIT"]
   262→}
   263→```
   264→
   265→### 7.2 Job Extraction
   266→
   267→AI helps extract structured job requirements.
   268→
   269→Example:
   270→
   271→```txt
   272→Part-Time Lecturer required for Community Health, Counselling Psychology, Social Work...
   273→```
   274→
   275→Becomes:
   276→
   277→```json
   278→{
   279→  "jobCategory": "EDUCATION",
   280→  "jobSubcategories": ["LECTURING", "TVET_TRAINING"],
   281→  "specializations": [
   282→    "COMMUNITY_HEALTH",
   283→    "COUNSELLING_PSYCHOLOGY",
   284→    "SOCIAL_WORK"
   285→  ]
   286→}
   287→```
   288→
   289→### 7.3 Normalization
   290→
   291→AI or parser logic maps different wording to the same taxonomy item.
   292→
   293→Examples:
   294→
   295→```txt
   296→Talent Acquisition → Recruitment
   297→People Operations → HR Operations
   298→Quick Books → QuickBooks
   299→Accounts Clerk → Accounting
   300→IT Helpdesk → IT Support
   301→```
   302→
   303→### 7.4 Transferable Skills Detection
   304→
   305→AI helps detect career transitions.
   306→
   307→Examples:
   308→
   309→```txt
   310→Cashier + CPA + reconciliation → Accounts Assistant
   311→Radio Presenter + Social Media → Communications Officer
   312→Enumerator + Data Collection → M&E Assistant
   313→```
   314→
   315→### 7.5 Explanation
   316→
   317→After the algorithm calculates the score, AI can explain the match.
   318→
   319→Example:
   320→
   321→```txt
   322→You are a strong match because your CV shows Accounts Payable, Bank Reconciliation, CPA progression, and financial reporting experience.
   323→
   324→Your main gap is SAP experience, which was not found in your CV.
   325→```

   377→## 9. Taxonomy Model
   378→
   379→The taxonomy is the foundation of the matching engine.
   380→
   381→It stores:
   382→
   383→- Organization types
   384→- Industries
   385→- Job categories
   386→- Subcategories
   387→- Roles
   388→- Skills
   389→- Tools
   390→- Qualifications
   391→- Certifications
   392→- Specializations
   393→
   394→### 9.1 taxonomy_items
   395→
   396→```sql
   397→CREATE TABLE taxonomy_items (
   398→  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
   399→
   400→  type ENUM(
   401→    'ORGANIZATION_TYPE',
   402→    'INDUSTRY',
   403→    'CATEGORY',
   404→    'SUBCATEGORY',
   405→    'ROLE',
   406→    'SKILL',
   407→    'TOOL',
   408→    'QUALIFICATION',
   409→    'CERTIFICATION',
   410→    'SPECIALIZATION',
   411→    'REGULATION',
   412→    'COMPETENCY'
   413→  ) NOT NULL,
   414→
   415→  value VARCHAR(120) NOT NULL,
   416→  label VARCHAR(255) NOT NULL,
   417→
   418→  parent_id BIGINT UNSIGNED NULL,
   419→
   420→  description TEXT NULL,
   421→
   422→  is_active BOOLEAN NOT NULL DEFAULT TRUE,
   423→
   424→  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
   425→  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
   426→
   427→  UNIQUE KEY unique_taxonomy_value (type, value),
   428→
   429→  INDEX idx_taxonomy_type (type),
   430→  INDEX idx_taxonomy_parent (parent_id),
   431→
   432→  CONSTRAINT fk_taxonomy_parent
   433→    FOREIGN KEY (parent_id)
   434→    REFERENCES taxonomy_items(id)
   435→    ON DELETE SET NULL
   436→);
   437→```
   438→
   439→Example rows:
   440→
   441→```txt
   442→CATEGORY: HUMAN_RESOURCES
   443→SUBCATEGORY: RECRUITMENT
   444→SKILL: CANDIDATE_SCREENING
   445→TOOL: QUICKBOOKS
   446→QUALIFICATION: BACHELOR_DEGREE
   447→INDUSTRY: HEALTHCARE
   448→```

  1789→### 18.2 Subcategory Match (✅ UPDATED)
  1790→
  1791→```js
  1792→function calculateSubcategoryMatch(candidate, job) {
  1793→  // Primary subcategory match = highest score
  1794→  if (candidate.primarySubcategoryId === job.primarySubcategoryId) {
  1795→    return 100;
  1796→  }
  1797→
  1798→  // Check candidate's extracted subcategories against job's subcategories
  1799→  const candidateSubcategories = candidate.subcategoryIds;
  1800→  const jobSubcategories = job.subcategoryIds;
  1801→
  1802→  if (!jobSubcategories.length) return 0;
  1803→
  1804→  const matched = jobSubcategories.filter(id =>
  1805→    candidateSubcategories.includes(id)
  1806→  );
  1807→
  1808→  const score = (matched.length / jobSubcategories.length) * 100;
  1809→
  1810→  // If candidate has it as an interest, boost slightly
  1811→  if (candidate.interestCategoryIds.includes(job.jobCategoryId)) {
  1812→    return Math.min(score + 10, 100);
  1813→  }
  1814→
  1815→  return score;
  1816→}
  1817→```

  1931→
  1932→  -- Category Score (Weight: 25%)
  1933→  CASE
  1934→    WHEN cp.primary_category_id = jp.job_category_id THEN 100
  1935→    WHEN EXISTS (
  1936→      SELECT 1
  1937→      FROM candidate_interests ci
  1938→      WHERE ci.candidate_id = ca.id
  1939→        AND ci.category_id = jp.job_category_id
  1940→    ) THEN 75
  1941→    ELSE 0

  1943→
  1944→  -- ✅ UPDATED: Subcategory Score (Weight: 25%)
  1945→  CASE
  1946→    -- Highest: Primary subcategory matches
  1947→    WHEN jp.primary_subcategory_id IS NOT NULL
  1948→      AND cp.primary_subcategory_id = jp.primary_subcategory_id

  1950→
  1951→    -- High: Candidate's extracted subcategories match job's subcategories
  1952→    WHEN EXISTS (
  1953→      SELECT 1
  1954→      FROM job_subcategories js
  1955→      INNER JOIN candidate_subcategories cs
  1956→        ON cs.candidate_id = ca.id
  1957→        AND cs.subcategory_id = js.subcategory_id
  1958→      WHERE js.job_id = j.id
  1959→    ) THEN 70
  1960→
  1961→    -- Medium: Candidate has the job's category as an interest
  1962→    WHEN EXISTS (
  1963→      SELECT 1
  1964→      FROM candidate_interests ci
  1965→      WHERE ci.candidate_id = ca.id
  1966→        AND ci.category_id = jp.job_category_id
  1967→    ) THEN 50
  1968→
  1969→    ELSE 0
  1970→  END AS subcategory_score,

  1972→  -- Skills Score (Weight: 25%)
